# Plan: Add Session Actions Menu to Chat Title Bar

## Context
The sidebar's `ProjectTree` component already has a dropdown menu for each session with "置顶", "重命名", "删除" actions. The user wants the same menu on the chat page title bar next to the session title, reusing components and logic.

## Approach: Extract a shared `SessionActionsMenu` component

### Step 1: Create `components/session-actions-menu.tsx`
A self-contained `"use client"` component that encapsulates:
- The `<DropdownMenu>` with `MoreHorizontal` trigger icon
- Pin/Unpin, Rename, Delete menu items (same icons: `Pin`, `Pencil`, `Trash2`)
- All three action handlers (API PATCH/DELETE + store `updateSession`/`removeSession`)
- Rename dialog state + `<Dialog>` UI (same pattern as project-tree.tsx lines 500-522)
- Delete confirmation dialog

Props: `{ projectId, sessionId, title, pinnedAt, className? }`

Internally uses `useAppStore()` for `updateSession`, `removeSession`, `currentSessionId`, and `useRouter()` for navigation on delete.

### Step 2: Update `components/sidebar/project-tree.tsx`
Replace the inline session `<DropdownMenu>` block (lines 425-489) and the rename `<Dialog>` (lines 500-522) with `<SessionActionsMenu>`. Remove the now-unused local state (`renameOpen`, `renameTarget`, `renameValue`) and handler functions (`handleTogglePin`, `openRenameDialog`, `handleRename`, `handleDeleteSession`). Keep `openMenuId` state if still needed for the project-level menu, or remove if only used for sessions.

### Step 3: Update `app/chat/[projectId]/page.tsx`
In the `<header>` (line 288), after the session title `<span>` (line 295), add:
```tsx
{currentSession && (
  <SessionActionsMenu
    projectId={projectId}
    sessionId={currentSession.sessionId}
    title={currentSession.title}
    pinnedAt={currentSession.pinnedAt}
    className="ml-1"
  />
)}
```
Import `SessionActionsMenu` at the top.
